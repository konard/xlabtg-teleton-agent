# Панель управления (Dashboard)

Dashboard — первая страница, которая открывается после входа. Это операторская «кабина»: текущий статус, активный провайдер и модель, счётчики токенов и стоимости, состояние сервисов, пользовательские виджеты и быстрые действия. Данные обновляются опросом каждые 10 секунд.

![Динамическая панель с виджетами](../assets/screenshots/ru/dynamic-dashboard-engine.png)

## Из чего состоит страница

Сверху вниз страница разделена на зоны:

| Зона | Содержимое |
| --- | --- |
| **Шапка статуса** | Провайдер, модель, аптайм, число сессий, число инструментов, суммы токенов, индикатор здоровья сервисов. |
| **Ряд быстрых действий (Quick Actions)** | Кнопки для безопасных частых операций: очистка кэша, экспорт логов, перезапуск агента, отправка тестового сообщения. Опасные действия требуют подтверждения. |
| **Полоса уведомлений** | Предупреждения (деградация провайдера, истёкший токен, отказы в вызове инструментов), информационные сообщения и неподтверждённые запросы на согласование. Сортируются по серьёзности. |
| **Сетка виджетов (Widget grid)** | Встроенные и сгенерированные виджеты. В режиме редактирования их можно перетаскивать и менять размер. |

Нижняя часть бокового меню всегда видна: она показывает, запущен ли runtime, позволяет переключать агентов, перезапускать runtime, менять тему и выходить из системы. Командная палитра (`Ctrl+K` / `Cmd+K`) доступна с любой страницы.

## Встроенные виджеты

«Из коробки» Dashboard содержит виджеты, покрывающие большинство задач:

- **Status** — провайдер, модель, fallback-модель, счётчики текущих задач.
- **Tokens & Costs** — расход токенов за день, прогноз стоимости за месяц, топ-модель по затратам.
- **Tool usage** — самые часто вызываемые и самые «падающие» инструменты со спарклайнами.
- **Activity heatmap** — тепловая карта активности по часам и дням недели.
- **Predictions** — превентивные подсказки в духе «инструмент X уперся в rate-limit — переключитесь на Y» или «очередь сессий растёт».
- **Cache** — процент попаданий, вытеснений, ключи-«тяжеловесы» и кнопка *Clear cache*.
- **Health checks** — состояние Telegram, LLM, MCP-серверов, векторного хранилища и TON-прокси.

## Режим редактирования и пользовательские виджеты

![Генератор виджетов](../assets/screenshots/ru/widget-generator.png)

Нажмите **Edit**, чтобы перейти в режим компоновки. В нём доступно:

1. **Перетаскивание** для изменения порядка и **изменение размера** за угловой маркер.
2. Кнопка **Add widget** — добавить готовый виджет из каталога.
3. Кнопка **Generate widget** — сформулировать виджет на естественном языке («показать инструменты, упавшие за неделю», «сравнить ежедневную стоимость токенов по провайдерам»). Генератор покажет превью, исходный запрос и компонент Recharts перед сохранением.
4. Иконка корзины — **удалить** виджет.
5. Кнопка **Done** — выйти из режима редактирования и сохранить раскладку.

Также можно **Export** — выгрузить пакет панели (раскладка в JSON + промпты генератора) и **Import** на другой установке.

> ℹ️ **Замечание.** Сгенерированные виджеты удобны для личных оперативных видов, но не являются источником истины. Для аудита, безопасности и конфигурации всегда обращайтесь к специализированным страницам: [Центр безопасности (Security Center)](08-security.md), [Аналитика (Analytics)](06-analytics.md), [Настройки (Configuration)](11-settings.md).

## Быстрые действия (Quick Actions)

| Действие | Что делает |
| --- | --- |
| **Clear cache** | Сбрасывает кэш запросов в памяти. Используйте после смены модели. |
| **Export logs** | Скачивает свежие логи в `.zip` для отправки в поддержку. |
| **Restart agent** | Пересоздаёт runtime; сохраняет Telegram-сессию и поставленные в очередь задачи. |
| **Send test message** | Отправляет одно сообщение в чат с администратором для проверки Telegram-пути. |

Любое разрушающее действие (Clear cache, Restart agent) открывает диалог *Confirm* с описанием. Если действие необратимо, потребуется ввести подтверждающее слово вручную.

## Ежедневный сценарий

1. **Взгляните на шапку статуса.** Провайдер, модель, индикатор «agent ready» должны быть зелёными.
2. **Прочтите баннеры.** Сначала закройте красные — обычно это просроченные credentials, отказы политик безопасности или ожидающие согласования.
3. **Пробегитесь по виджетам Tokens, Tools, Activity.** Резкий всплеск чаще всего связан с одной автономной задачей или зацикленным процессом.
4. **При необходимости запустите быстрые действия** — очистите кэш после смены модели, выгрузите логи для расследования инцидента.
5. **Откройте профильные страницы**, если нужно глубже: [Аналитика (Analytics)](06-analytics.md) для трендов, [Tasks](10-advanced-features.md#tasks) для глубины очереди, [Security](08-security.md) для журнала аудита.

## Полезные привычки

- Держите одну **компактную панель** для повседневной работы и отдельную — для диагностики.
- Карточки predictions и anomaly воспринимайте как **подсказки**, а не как разрешения; перед действием убедитесь в исходных данных.
- После любого быстрого действия, меняющего состояние (restart, clear cache), откройте Security Center и проверьте, что соответствующее событие появилось в журнале аудита.
- Сохраняйте свои пользовательские раскладки виджетов через Export — это позволит восстановить их после переустановки.
