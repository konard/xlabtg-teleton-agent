# Аналитика (Analytics)

Страница Analytics нужна, чтобы понимать, как агента используют, куда уходят деньги и здорово ли работает система. Это многовкладочный дашборд с графиками, таблицами и фильтрами на базе Recharts.

![Аналитика временного контекста](../assets/screenshots/ru/temporal-context-analytics.png)

## Структура страницы

В шапке — селектор периода и кнопка экспорта. Ниже — лента вкладок:

| Вкладка | На чём фокус |
| --- | --- |
| **Tokens** | Расход токенов по дням, моделям, провайдерам и сессиям. |
| **Costs** | Деньги: по провайдерам, моделям, инструментам. Включает индикатор бюджета. |
| **Tools** | Самые вызываемые и самые «падающие» инструменты, success rate, среднее время. |
| **Activity** | Тепловая карта объёма сообщений и tool-вызовов по часам и дням. |
| **Performance** | Задержки p50/p95/p99, доля ошибок, глубина очередей. |
| **Anomaly** | Автоматически найденные необычные события с уровнем серьёзности и контекстом. |
| **Temporal** | Долгосрочные повторяющиеся паттерны (часы пик, недельные циклы). |

**Селектор периода** — `1h`, `24h`, `7d`, `30d` или произвольный диапазон. **Export** скачивает данные текущей вкладки в JSON или CSV.

## Tokens

Вкладка Tokens содержит три представления:

- **Daily totals** — линейный график input + output токенов; режим stacked area сравнивает по провайдерам.
- **By model** — столбчатая диаграмма расхода токенов по моделям. При наведении показывает число вызовов.
- **Top sessions** — таблица самых «токеноёмких» сессий за период; клик по строке переводит на [Сессии (Sessions)](07-sessions.md).

Открывайте вкладку, когда:

- Виджет токенов на Dashboard показывает скачок — переключитесь на 24h и найдите модель/сессию-виновника.
- Тестируете новую модель — сравните одну и ту же группу задач за неделю до и после смены.

## Costs

Вкладка Costs добавляет денежную ось поверх данных по токенам:

- **Cumulative spend** за выбранный период.
- **Projection** — экстраполяция на конец календарного месяца (после накопления достаточного количества samples).
- **Budget bar** — текущий расход относительно бюджета из [Настроек (Configuration)](11-settings.md). Жёлтый при 70%, красный при 90%.
- **Cost by tool** — полезно, когда внешний API-инструмент неожиданно «съедает» бюджет.

> ℹ️ **Замечание.** Стоимость считается по таблицам цен моделей. Self-hosted провайдеры по умолчанию `$0`. Если ваша реальная стоимость отличается, поправьте таблицу цен в [Configuration → LLM](11-settings.md).

## Tools

Вкладка Tools влияет на решения, принимаемые на странице [Tools](04-tools.md):

- **Top by calls** — столбчатая диаграмма самых вызываемых инструментов.
- **Top by failures** — отсортирована по числу ошибок, рядом success rate.
- **Latency** — таблица с средней длительностью и p95.
- **Unused** — список включённых инструментов, которые ни разу не вызывались за период.

Используйте еженедельно, чтобы находить кандидатов на отключение (unused, всегда падающие) или на сужение scope (admin-only после кластера ошибок).

## Activity

Activity — тепловая карта сумм событий «час × день недели»: входящих сообщений, исходящих ответов, tool-вызовов, событий автономного режима. Чем ярче ячейка, тем выше активность.

Используйте, чтобы:

- Запланировать heartbeat / digest задачи на «тихие» часы.
- Обосновать увеличение параллельности на пиках.
- Увидеть разницу между будни/выходные.

## Performance

Вкладка Performance показывает задержки и надёжность:

- **End-to-end response time** — линии p50, p95, p99.
- **Tool latency** — p95 по каждому инструменту.
- **Error rate** — процент неуспешных ответов.
- **Queue depth** — глубина очередей автономных задач и входящих сообщений.

Регрессия здесь обычно означает проблемы провайдера, смену модели или зацикленную автономную задачу. Прежде чем писать вендору, проверьте [Tasks](10-advanced-features.md#tasks) на «зависшие» задачи.

## Anomaly

![Мониторинг аномалий](../assets/screenshots/ru/anomaly-monitoring.png)

На вкладке Anomaly — автоматически найденные события с тремя уровнями: `info`, `warning`, `critical`. Каждая карточка содержит:

- **What changed** (например, рост токенов +320% неделя к неделе).
- **Suspected cause** — модель, инструмент, автономная задача или оператор.
- **Recommended action** — ссылка на страницу, где это чинится.
- **Acknowledge** — оператор подтверждает, что аномалию посмотрели.

Подтверждайте аномалию только после того, как поняли причину; неподтверждённая остаётся вверху и поднимает красный баннер на Dashboard.

## Temporal

Вкладка Temporal отслеживает длинные паттерны:

- **Peak hours by day of week** — для планирования.
- **Recurring incident windows** — окна, где раз за разом случаются ошибки.
- **Drift** — медленные тренды стоимости и задержек, которые ежедневный Dashboard не ловит.

## Обучение на обратной связи

![Дашборд обучения по обратной связи](../assets/screenshots/ru/feedback-learning-dashboard.png)

Вкладка Tokens ведёт на страницу [Feedback](10-advanced-features.md#feedback) с собственным аналитическим видом. Метрики обратной связи:

- **Средний рейтинг** во времени.
- **Доля положительных и отрицательных** ответов.
- **Themes** — кластеры содержания (например, «слишком формальный тон», «потерял контекст»).
- **Recent feedback** — хронологический список с рейтингом, тоном и сообщением, которое вызвало оценку.

Метрики обратной связи используйте вместе с экспериментами в [Редакторе Soul](05-soul-editor.md), когда меняете тон или стиль ответа.

## Экспорт данных

Кнопка экспорта скачивает данные текущей вкладки в JSON (полная структура) или CSV (строки). Применения:

- **Ревизии стоимости** — сравнение «месяц к месяцу» в таблицах.
- **Отчёты об инцидентах** — сырьё к описанию.
- **Долговременная архивация** — при ротации SQLite-базы.

> ⚠️ **Считайте экспорты чувствительными** — там операционные тайминги, метаданные чатов и имена инструментов. Храните рядом с конфигом агента.

## Чек-лист еженедельного обзора

- Рост токенов соответствует реальной нагрузке, а не зацикленному циклу.
- Ошибки инструментов не сосредоточены в одном модуле.
- Прогноз стоимости не превышает заданный бюджет.
- У каждой аномалии есть владелец и объяснение; либо acknowledge, либо эскалация.
- Тематики из feedback отражены либо в правке промпта ([Редактор Soul](05-soul-editor.md)), либо в правке политики ([Security Center](08-security.md)).
