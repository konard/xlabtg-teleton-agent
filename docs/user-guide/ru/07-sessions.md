# Сессии (Sessions)

Страница Sessions — это поисковый архив всех чатов, к которым прикасался агент. Открывайте её, когда пользователь спрашивает «почему агент так ответил», когда аудиту нужны доказательства, когда вы расследуете инцидент в конкретном чате.

![Self-correction monitoring](../assets/screenshots/ru/self-correction-monitoring.png)

## Структура страницы

Страница разделена по горизонтали:

- **Левая колонка — список сессий.** Каждая строка содержит заголовок чата (или имя + фамилию для DM), бейдж типа (`dm`, `group`, `channel`, `bot`), превью последнего сообщения и время последней активности. Сверху — поле поиска по заголовку, username и тексту сообщений; выпадающий фильтр сужает по типу чата.
- **Правая колонка — детали сессии.** Пустая, пока не выбрана сессия. Показывает шапку с метаданными, полный список сообщений и панель коррекций.

## Список сессий

В строке списка отображаются:

| Поле | Описание |
| --- | --- |
| **Chat title** | Имя группы, канала или контакта. |
| **Chat type** | `dm` (личное), `group`, `channel` или `bot`. |
| **Last message** | Превью последнего сообщения, обрезано. |
| **Last activity** | Время в локальном формате. |
| **Provider / model** | Модель, обработавшая последний ответ. |

Поиск ищет по заголовку чата, Telegram-username и тексту сообщений. Фразу в кавычках ищет точно. Фильтр chat-type позволяет ограничить поиск (например, «`buy`» только в DM).

## Просмотр сессии

Откройте сессию щелчком по строке. В правой панели:

- **Шапка** — chat ID, заголовок, username, тип чата, провайдер, модель, число сообщений, суммарный input/output по токенам, время начала и последнего обновления.
- **Messages** — хронологический список. Каждое сообщение помечено источником (`user`, `agent`, `system`, `tool`), показывает наличие медиа, факт редактирования, время, отношения reply и (для ответов агента) модель, которая его сгенерировала.
- **Tool calls** — встроенные блоки внутри сообщений агента: имя инструмента, аргументы, сокращённый результат.
- **Corrections** — если сработала самокоррекция, у сообщения появляется бейдж *corrected*; раскройте, чтобы увидеть исходный и итоговый текст.

Действие **Copy** на уровне сообщения копирует одно сообщение; **Export** на уровне сессии скачивает полный диалог в Markdown или JSON.

## Восстановление контекста

Когда пользователь спрашивает «почему агент ответил X»:

1. Найдите сессию по username или заголовку чата.
2. Откройте её и прокрутите до спорного сообщения.
3. Прочитайте **предыдущее сообщение пользователя** — это и есть запрос, на который реагировал агент.
4. Раскройте все **tool calls** между сообщением пользователя и ответом агента — это факты, на которые опирался ответ.
5. Если была коррекция, прочитайте **original output** и **reflection** — увидите, почему цикл предпочёл корректуру.

Только пройдя эту цепочку, имеет смысл думать про правки в [Редакторе Soul](05-soul-editor.md) или политики в [Центре безопасности (Security Center)](08-security.md).

## Самокоррекции

![Контекст обучения по обратной связи](../assets/screenshots/ru/feedback-learning-dashboard.png)

*Correction* — запись, когда цикл самокоррекции заменил исходный ответ. В записи:

- **Original output** — что сгенерировал агент сначала.
- **Evaluation score** — оценка цикла.
- **Reflection** — объяснение на естественном языке, почему нужен переписать.
- **Corrected output** — то, что увидел пользователь.
- **Escalation state** — понадобилось ли человеческое подтверждение (ссылка на [Security Center](08-security.md)).
- **Tool recovery guidance** — что сделали при сбое инструмента.

Паттерны, на которые стоит смотреть:

- Частые коррекции в одном типе чата — обычно несоответствие тона или контекста: правьте [Редактор Soul](05-soul-editor.md).
- Коррекции, сосредоточенные вокруг одного инструмента — нестабильность инструмента: откройте [Tools](04-tools.md) и посмотрите failure rate.
- Повторные эскалации на одной и той же цели — слишком жёсткая политика или плохо ограниченная цель: пересмотрите [Автономный режим](03-autonomous-mode.md).

## Встроенная обратная связь

Каждое сообщение агента имеет иконки thumbs-up / thumbs-down. Оператор может оценить ответ прямо в чате. Оценки попадают в [Feedback](10-advanced-features.md#feedback) и в feedback-вид [Аналитики](06-analytics.md).

> ℹ️ **Замечание.** Встроенная обратная связь — только на стороне оператора. Конечные пользователи иконок не видят, они остаются в WebUI.

## Экспорт сессии

Кнопка **Export** в шапке деталей скачивает:

- **Markdown** — читаемый человеком транскрипт.
- **JSON** — полная структурированная нагрузка с tool calls, идентификаторами моделей и временами.

> ⚠️ **Считайте экспорты чувствительными** — там тексты сообщений, Telegram-usernames, метаданные медиа и решения оператора. Никогда не вставляйте их в публичные трекеры. Делитесь по тем же каналам, что и credentials.

## Хорошие практики

- **Фильтруйте по типу чата** до широкого ревью — у DM и групп разное соотношение сигнал/шум.
- **Ищите словами пользователя**, а не внутренними обозначениями — то, как пользователь написал, важнее того, как агент это классифицировал.
- **Экспортируйте только нужные сессии** для расследования; не выгружайте «на всякий случай».
- **Не вставляйте экспорты** в публичные трекеры, GitHub PR или каналы Slack, видимые посторонним.
- Если сессия выявила пробел в политике — сначала правьте [Security Center](08-security.md) (жёсткое правило), потом [Редактор Soul](05-soul-editor.md) (мягкое правило).
