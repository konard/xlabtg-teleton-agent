# Инструменты (Tools)

Страница Tools управляет тем, какие встроенные инструменты доступны агенту и где каждый из них разрешено вызывать. Вместе с [Центром безопасности (Security Center)](08-security.md) это главная поверхность безопасности в WebUI: всё, что вы здесь выключите, не сможет вызвать ни LLM, ни автономная задача, ни удалённый агент.

![Виджеты-каталог инструментов на панели управления](../assets/screenshots/ru/dynamic-dashboard-engine.png)

## Из чего состоит страница

Страница — это сгруппированный список, по строке на каждый инструмент, сгруппированный по **модулям**. Каждая строка содержит:

- **Имя** инструмента и краткое описание.
- Переключатель **Enabled**.
- Селектор **Scope** (`always`, `dm-only`, `group-only`, `admin-only`).
- **Cost badge** — `low`, `medium`, `high` — оценка задержки, платной стоимости API или операционного риска.
- **Stats**: общее число вызовов, число успехов, число ошибок, дата последнего использования, средняя длительность.
- Стрелку **expand**, открывающую панель деталей.

Над списком:

- **Поле поиска** — ищет по имени, описанию, параметрам и модулю.
- **Фильтр состояний** — *all / enabled / disabled*.
- **Sort** — *by module* (удобно для аудита) или *by name* (когда точно знаете нужный инструмент).
- **Bulk actions** — чекбоксы выделяют несколько инструментов; панель массовых действий появляется сверху.
- **Import / Export** — JSON-снимки полного состояния enable/scope.

## Понятия

| Понятие | Описание |
| --- | --- |
| **Module** | Группа связанных инструментов. Встроенные модули: Telegram, TON, web, workspace, exec, memory. Манифесты плагинов добавляют свои модули. |
| **Enabled** | Если выключено, инструмент невидим для LLM и для автономного планировщика — он не появляется в схеме function-calling. |
| **Scope** | Где инструмент может выполняться. `always` — в любом чате. `dm-only` — только личные сообщения. `group-only` — только в группах. `admin-only` — только админские чаты, независимо от типа. |
| **Cost badge** | Грубая оценка: low — локально и быстро, medium — платный API, high — он-чейн или разрушающие операции. Полезно при массовом отключении. |
| **Stats** | Счётчики вызовов и ошибок, дата последнего использования, средняя длительность. Страница опрашивает их живьём — сразу видно проблемные инструменты. |

## Просмотр инструмента

Кликните по строке — откроется панель деталей:

- **Сигнатура функции** — имя, список параметров, типы, тип возвращаемого значения.
- **Документация** — markdown-описание из манифеста с примерами.
- **Recent failures** — последние ошибки с временем, аргументами и сообщением.
- **Test panel** — заполните параметры и нажмите **Run**, чтобы вызвать инструмент прямо из WebUI. Тестовые вызовы привязаны к аккаунту оператора и попадают в журнал аудита.

> ⚠️ **Тестируйте с осторожностью.** Кнопка `Run` в тест-панели — это реальный вызов. Для Telegram и TON начинайте с тестового аккаунта и небольшого количества TON. Workspace и `exec`-инструменты тестируйте только в одноразовой директории.

## Включение и установка scope

1. Найдите инструмент через поиск или раскройте модуль.
2. Включите переключатель **Enabled**. (Сохранение происходит автоматически с дебаунсом.)
3. Выберите **самый строгий приемлемый scope**:
   - `admin-only` — для всего, что переводит средства, пишет файлы или запускает shell-команды.
   - `dm-only` — для чувствительных сценариев поддержки, где оператор работает с пользователем «один на один».
   - `group-only` — для модерации и аналитики групп.
   - `always` — только для read-only и не приводящих к расходам инструментов.
4. Если инструмент чувствительный, откройте [Центр безопасности (Security Center)](08-security.md) и убедитесь, что в журнале аудита появилось событие `tool_toggle`, а лог валидации обрабатывает новый scope ожидаемо.

## Высокорисковые модули

В установке по умолчанию эти модули выключены или ограничены до admin-only. Не меняйте это без конкретной причины и соответствующей политики в Security Center:

- **`exec`** — запускает shell-команды на хосте. По умолчанию выключен. Включайте только с scope `admin-only` и совместно с политикой `exec` в Security Center.
- **TON-инструменты кошелька** (`ton_send`, `jetton_send`) — держите admin-only и за политикой `require_approval`.
- **Workspace write/delete** — ограничьте доверенными операторами; для `delete` рекомендуется политика согласования.
- **Внешние HTTP-инструменты** — пересмотрите список целевых доменов; если инструмент может изменять данные, ставьте scope `admin-only`.

## Массовые операции

Чекбоксами в строках выделите несколько инструментов и используйте плавающую панель действий:

| Массовое действие | Сценарий |
| --- | --- |
| **Toggle enabled** | Отключить целый модуль перед security-ревью или во время инцидента. |
| **Set scope** | Перевести все TON send-инструменты в `admin-only` одним щелчком. |
| **Export selected** | Снять снимок перед крупным изменением. |
| **Import** | Применить заведомо безопасную конфигурацию к свежей установке. |

> ℹ️ **Замечание.** Import перезаписывает только enable-state и scope; он не устанавливает новые инструменты. Plugin-инструменты приходят со страницы [Плагины (Plugins)](10-advanced-features.md#plugins).

## Plugin-инструменты

Плагинные инструменты появляются в том же списке, что и встроенные, и помечены идентификатором плагина. Поведение идентичное: тот же переключатель, тот же scope, те же события аудита. Но они приходят из манифеста плагина. Перед включением плагина в продакшене:

1. Прочитайте манифест на странице [Plugins](10-advanced-features.md#plugins). Проверьте `requested_permissions`, `requested_secrets` и URL источника.
2. Убедитесь, что у автора плагина есть значок верификации (Plugins → Marketplace).
3. Сначала протестируйте в DM с маленьким входом, прежде чем разрешать groups.

## Рекомендуемый аудит

- **Еженедельно** — сортировать по модулю, искать инструменты с большим числом ошибок; разбираться или отключать.
- **После смены модели** — очистить кэш на [Dashboard](02-dashboard.md), затем прогнать включённые инструменты через тест-панель.
- **Перед делегированием удалённому агенту** — открыть [Network](10-advanced-features.md#network) и убедиться, что делегату видны только намеренно выставленные инструменты.
- **После инцидента** — выгрузить конфигурацию инструментов через Export и приложить снимок к отчёту инцидента.
