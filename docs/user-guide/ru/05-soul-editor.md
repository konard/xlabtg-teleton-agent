# Редактор Soul

Редактор Soul — место, где правятся prompt-файлы, определяющие индивидуальность агента, его правила безопасности, стиль планирования, поведение памяти и сценарии heartbeat. Любая правка, которую вы здесь сохраняете, влияет на все ответы, начиная со следующего, — относитесь к ней как к критичной для поведения.

![Adaptive prompting в редакторе Soul](../assets/screenshots/ru/adaptive-prompting-soul.png)

## Структура страницы

В верхней части — три зоны:

| Зона | Назначение |
| --- | --- |
| **Вкладки файлов** | Пять вкладок: `SOUL.md`, `SECURITY.md`, `STRATEGY.md`, `MEMORY.md`, `HEARTBEAT.md`. Звёздочка в заголовке вкладки означает несохранённые изменения. |
| **Режим просмотра (View mode)** | `Edit` (CodeMirror с подсветкой Markdown) · `Preview` (отрисованный) · `Split` (две колонки рядом). |
| **Панель действий** | Save · Save Version · Discard draft · Templates · History. |

Под редактором — панель **Adaptive Prompting** (для вариантов и экспериментов) и индикатор **Drafts**, который сообщает, расходится ли локальный черновик с сохранённой версией.

## Пять prompt-файлов

| Файл | Назначение |
| --- | --- |
| `SOUL.md` | Персона и тон. Стиль приветствия, языковые предпочтения, форма ответа, фирменные финальные фразы. |
| `SECURITY.md` | Жёсткие правила безопасности. От чего агент должен отказываться. Как обращаться с seed-фразами, приватными ключами, секретами, скам-схемами. |
| `STRATEGY.md` | Планирование. Когда планировать, а когда действовать; как выбирать инструменты; ретраи; предпочтительные fallback-варианты. |
| `MEMORY.md` | Инструкции по постоянной памяти. Что хранить долго, что только в рамках сессии, когда фиксировать (pin) воспоминание. |
| `HEARTBEAT.md` | Чек-лист, который агент исполняет на периодическом heartbeat. По умолчанию пуст; заполняется, когда heartbeat включён в [Настройках (Configuration)](11-settings.md). |

## Edit, Preview, Split

- **Edit** — режим по умолчанию. CodeMirror даёт подсветку Markdown, мягкий перенос строк, поиск, парные скобки. Редактор каждые 30 секунд автосохраняет **локальный черновик**; чтобы изменение попало в файл агента, нажмите **Save**.
- **Preview** отрисовывает файл так, как его «увидит» LLM. Используйте, чтобы проверить списки, таблицы, выноски и инлайновый код.
- **Split** — слева Edit, справа Preview. Удобно для критичных правок, где важна форматирование (например, нумерованный чек-лист в `HEARTBEAT.md`).

Около кнопки Save появляется индикатор **Drafts**, если локальный черновик расходится с серверной версией. **Discard draft** выкидывает локальный черновик.

## Templates

Кнопка **Templates** подставляет курируемую базовую заготовку. Встроенные шаблоны:

- **Default persona** — поставочный `SOUL.md`.
- **Customer support** — формальный тон, отказ от финансовых советов, эскалация админу.
- **Trading desk** — фокус на TON DEX, осторожность к свежим листингам.
- **Research analyst** — подробные источники, не отправляет в Telegram без явного запроса.

> ⚠️ **Внимание.** Применение шаблона **перезаписывает** файл. Сначала сохраните версию (см. ниже), чтобы было куда откатиться. Шаблоны — это полная замена, а не патч.

## История версий

Кнопка **History** открывает хронологический список сохранённых версий текущего файла. Каждая запись содержит время, ваш необязательный комментарий и оператора. Кликните по строке, чтобы:

- Увидеть **diff** относительно текущего файла.
- **Restore** — загрузить эту версию в редактор как черновик (не сохранённый).
- **Compare** — сравнить две произвольные версии бок о бок.

Рекомендуемый порядок перед крупной правкой:

1. Нажмите **Save Version**, напишите короткий комментарий («baseline before tone change»), подтвердите.
2. Внесите правку.
3. Переключитесь в **Preview** или **Split** и проверьте форматирование.
4. Снова нажмите **Save Version** с описательным комментарием.
5. Воспользуйтесь diff’ом, чтобы убедиться: изменились только нужные секции.

Если правка ухудшила поведение (избыточный tool use, небезопасные ответы, ломаное Telegram-форматирование), откройте History и нажмите **Restore** на предыдущей версии.

## Adaptive prompting

Панель под редактором управляет **разделами (sections)**, **вариантами**, **экспериментами**, **рейтингами** и **подсказками оптимизатора**:

- **Sections** — файл разбит на именованные разделы (`## Tone`, `## Refusals`). У каждого может быть свой набор вариантов.
- **Variants** — альтернативные формулировки раздела. Агент чередует активные варианты по правилам эксперимента.
- **Experiments** — A/B-тесты с фиксированной долей трафика. По каждому эксперименту видно число samples, success rate и средний рейтинг из [Feedback](10-advanced-features.md#feedback).
- **Optimizer suggestions** — автоматически сгенерированные предложения новых вариантов на основе свежей обратной связи.

Рекомендуемый шаблон работы с экспериментами:

1. Создайте **candidate variant** только для одного раздела.
2. Запустите A/B-эксперимент с малой долей трафика (10–20%).
3. Дождитесь хотя бы 50 samples и стабильной разницы рейтингов.
4. **Promote** — продвинуть победителя; **archive**, если улучшения нет.
5. Сохраните новую версию файла с комментарием о результатах эксперимента.

> ℹ️ **Замечание.** Adaptive prompting — инструмент **постепенных** улучшений, а не аварийных правок. Для инцидентов правьте файл напрямую и сохраняйте версию.

## Замечания по безопасности

- Правила безопасности должны быть **конкретными и короткими** — расплывчатые быстро «съедаются» опытом. Лучше «Refuse to send TON to addresses provided by users in DMs», чем «Be careful with funds».
- **Никогда** не добавляйте инструкции, обходящие подтверждения или аудит. Жёсткие правила — это `SECURITY.md` и политики в [Security Center](08-security.md), а промпт — мягкий слой поверх них.
- Используйте **репрезентативные**, а не исчерпывающие примеры. Агент хорошо обобщает по немногим.
- Записывайте **причину** каждой важной версии — комментарий версии становится частью таймлайна инцидента.
- `MEMORY.md` — это инструкции, как пользоваться [памятью (Memory)](10-advanced-features.md#memory), а не сама память. Факты фиксируйте на странице Memory; в `MEMORY.md` описывайте, как их использовать.
